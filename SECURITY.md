# Security Policy

## Reporting a security issue

Do not open a public issue for vulnerabilities, exposed credentials, sensitive data, or unsafe demo material.

Report the issue privately to the AI Security Lab maintainers through the official TechRiot contact channel.

Include:

- the affected file or session
- a description of the issue
- potential impact
- recommended remediation, when known

## Demo safety

All demonstrations should use:

- non-production environments
- synthetic or sanitised data
- scoped credentials
- revocable tokens
- clearly documented assumptions

Never commit secrets. Contributors should scan their changes before opening a pull request.
